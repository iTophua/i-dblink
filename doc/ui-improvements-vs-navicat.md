# iDBLink UI/交互完善文档（对照 Navicat Premium）

> 本文档用于指导其他 AI 进行开发，详细列出当前 iDBLink 与 Navicat Premium 在 UI 和交互层面的差距，按模块分类，每个条目包含「Navicat 标准做法」「当前状态」「需完善点」「优先级」。

---

## 一、整体布局与导航

### 1.1 主窗口框架

| 维度 | Navicat Premium | 当前 iDBLink | 需完善点 | 优先级 |
|------|----------------|-------------|---------|--------|
| **标题栏** | 显示当前活跃连接名+数据库名，如 `localhost (mydb) - Navicat` | 无动态标题，固定为应用名 | 根据当前活跃 Tab 的连接和数据库动态更新窗口标题 | 低 |
| **菜单栏（Menu Bar）** | 完整的原生菜单栏（文件/编辑/查看/工具/窗口/帮助），含二级菜单和快捷键提示 | 仅 Toolbar 上的下拉按钮模拟菜单，无原生菜单栏（Tauri 侧有但未完全对接） | 在 Tauri 侧完善原生菜单栏，确保 macOS/Windows 都有标准菜单体验；当前 Toolbar 的模拟菜单保留作为备选 | 中 |
| **工具栏（Toolbar）** | 可自定义的工具栏按钮，支持显示/隐藏图标+文字、仅图标、仅文字 | 固定样式的文本按钮，不可自定义 | 增加工具栏自定义配置：显示模式切换、按钮增删、分组分隔线 | 低 |
| **状态栏（Status Bar）** | 底部状态栏显示：当前连接状态、服务器版本、编码、事务状态、行数统计、执行时间等 | 有 StatusBar 组件但信息极少，仅显示连接名和表名 | 丰富状态栏内容：显示当前连接的数据库类型版本、字符集、当前选中行数/总记录数、事务状态（自动提交/手动）、最后操作耗时 | 中 |
| **快捷启动栏** | 左侧可固定的「收藏夹」或「快捷方式」面板 | 无 | 可在侧边栏顶部增加快捷操作区（新建查询、打开表等常用操作） | 低 |

### 1.2 工作区布局

| 维度 | Navicat Premium | 当前 iDBLink | 需完善点 | 优先级 |
|------|----------------|-------------|---------|--------|
| **标签页（Tab）系统** | 支持多行标签页、标签页分组、标签页拖拽重新排序、标签页拖拽到独立窗口 | 单行标签页，支持关闭/右键菜单，但不支持拖拽排序和独立窗口 | 1. 实现标签页拖拽排序；2. 支持标签页拖拽脱离为独立浮动窗口（Tauri 多窗口）；3. 标签页过多时支持多行或滚动+下拉列表 | 高 |
| **窗口分割** | 支持同一查询结果的水平/垂直分割查看 | 无 | SQL 编辑器结果面板支持左右/上下分屏查看多个结果集 | 低 |
| **面板隐藏/显示** | 所有面板（对象窗格、属性窗格、SQL 编辑器、结果窗格）均可收起/展开 | 侧边栏可收起，但 SQL 编辑器结果面板只能拖拽调整高度，无法完全隐藏 | 增加面板的显隐切换按钮（如结果面板的「收起/展开」按钮） | 中 |

## 二、连接管理（Connection Management）

### 2.1 连接对话框

| 维度 | Navicat Premium | 当前 iDBLink | 需完善点 | 优先级 |
|------|----------------|-------------|---------|--------|
| **数据库类型选择** | 左侧分类列表（常用/云端/文件型/国产等），点击后右侧动态切换配置表单 | 已实现左侧分类+右侧配置，UI 结构类似 | ✅ 已较完善，但需补充更多国产数据库（如 TiDB、OceanBase、PolarDB） | 中 |
| **连接配置字段** | 根据数据库类型动态显示字段：MySQL 有认证插件、PostgreSQL 有 SSL 模式/服务名、SQLite 直接文件选择器 | SQLite 用文本输入路径，其他基本字段有 | 1. SQLite 增加文件选择器按钮（调用 Tauri dialog API）；2. 各数据库类型增加特有高级字段（如 MySQL 的 `auth_plugin`、PG 的 `sslmode`、SQL Server 的 `instance`）；3. 增加「使用密码」选项：Prompt(提示)/Use SSH key/Use authentication等 | 高 |
| **连接测试** | 点击测试后显示详细错误信息（网络不通/认证失败/权限不足等），有加载动画 | 有测试按钮和加载状态，但错误信息较笼统 | 细化错误分类和提示：区分网络超时、认证失败、数据库不存在、权限拒绝等场景 | 中 |
| **连接颜色分类** | 可为连接设置颜色标签，连接名在树中显示对应颜色 | 分组有颜色，但单个连接无颜色标签 | 增加连接级别的颜色标签设置，在连接树中显示 | 低 |
| **连接分组管理** | 支持嵌套分组、拖拽连接到分组、分组排序 | 支持分组，但无嵌套分组，拖拽移动较基础 | 1. 支持分组嵌套（当前 group 表有 parent_id 字段但 UI 未实现嵌套展示）；2. 支持拖拽连接到分组（当前通过右键菜单移动，需增加拖拽交互） | 中 |
| **保存密码策略** | 可选择「保存密码在钥匙串」「不保存」「使用 SSH 密钥」等 | 保存到系统密钥链，但无策略选项 | 增加密码保存策略选择：Prompt(每次询问)/Save Password in Keychain/Use SSH Public Key等 | 中 |

### 2.2 连接树交互

| 维度 | Navicat Premium | 当前 iDBLink | 需完善点 | 优先级 |
|------|----------------|-------------|---------|--------|
| **连接状态指示** | 连接中/已连接/断开三种状态，带不同颜色图标和动画 | 有状态圆点（绿色=已连接，黄色=连接中），但断开状态不明显 | 增加断开状态的灰色/虚线图标，未保存密码的连接显示钥匙图标 | 低 |
| **数据库展开行为** | 展开连接后列出所有数据库，数据库图标区分系统库/用户库 | 展开后列出数据库，但无系统库/用户库区分 | 增加数据库图标区分：系统数据库（如 mysql、information_schema）用不同图标或置灰显示 | 低 |
| **对象类型层级** | 连接 > 数据库 > 对象类型（表/视图/函数/存储过程/触发器/事件）> 具体对象 | 连接 > 数据库 > 表/视图/存储过程/函数（存储过程和函数是占位符，未实现） | 1. 实现存储过程、函数的列表加载和展示；2. 增加触发器（Trigger）节点；3. 增加事件（Event）节点（MySQL）；4. 对象类型节点应支持右键新建对应类型对象 | 高 |

| **拖拽操作** | 支持：表拖拽到查询编辑器生成 SELECT、拖拽到另一个数据库复制表、连接拖拽排序 | 无拖拽操作 | 1. 表节点拖拽到 SQL 编辑器生成 `SELECT * FROM table`；2. 表节点拖拽到另一个数据库节点触发复制/移动对话框 | 低 |
| **批量操作** | 树中支持 Ctrl/Cmd 多选对象，右键批量导出/删除 | 单选模式 | 支持多选对象（Shift/Ctrl 连续/多选），右键批量操作 | 低 |

## 四、数据表浏览与编辑（Data Grid）

### 4.1 数据网格核心功能

| 维度 | Navicat Premium | 当前 iDBLink | 需完善点 | 优先级 |
|------|----------------|-------------|---------|--------|
| **分页机制** | 底部状态栏分页：显示当前页/总页、每页行数选择（100/500/1000/全部）、跳转页码输入 | 有分页组件（currentPage + pageSize），但每页大小不可选，无总页数显示 | 1. 增加每页行数选择器（10/50/100/500/1000/5000/全部）；2. 显示总页数；3. 增加页码跳转输入框 | 中 |

| **外键关联查看** | 点击外键列的值可跳转到引用表，或显示引用行详情 | 无 | 外键列的值显示为蓝色链接，点击可：1. 在新 Tab 打开引用表并定位到对应行；2. 或弹出小窗口显示引用行数据 | 中 |
| **行详情面板** | 底部或侧边可展开行详情面板，以表单形式展示当前行的所有字段（适合宽表） | 无 | 增加「行详情」底部面板，点击行后自动展开，显示当前行字段的表单视图 | 低 |
| **图像/二进制预览** | BLOB 字段可预览图片、Hex 模式、保存到文件 | 无 | BLOB/BINARY 字段增加预览按钮：图片预览、Hex 编辑器、导出到文件 | 低 |
| **数据搜索** | 支持在结果集中全局搜索（非数据库查询），高亮匹配项 | 有 Quick Filter（AG Grid 的客户端过滤） | 增强全局搜索：支持正则、高亮、逐行跳转 | 低 |

## 五、SQL 编辑器（Query Editor）

### 5.1 编辑器核心

| 维度 | Navicat Premium | 当前 iDBLink | 需完善点 | 优先级 |
|------|----------------|-------------|---------|--------|
| **编辑器基础** | Monaco 风格的编辑器，支持语法高亮、折叠、多光标、列选择模式 | 使用 Monaco Editor，基础功能已有 | ✅ 已较完善 | - |
| **智能补全** | 根据上下文补全：表名、列名、函数、关键字；支持数据库/Schema 前缀 | 已实现基础补全（关键字、函数、表名、列名），有上下文分析 | 1. 补全应包含 Schema/数据库前缀（如 `db.table.column`）；2. 函数补全应显示参数提示（signature help）；3. 增加别名补全（如 `SELECT a. FROM table AS a` 应提示 a 的列） | 高 |
| **代码格式化** | 一键格式化 SQL，支持多种格式化风格配置 | 有 formatSQL 按钮，但实现较简单（仅大写关键字+换行） | 1. 接入专业 SQL 格式化库（如 `sql-formatter`）；2. 支持格式化配置：关键字大小写、缩进风格、换行规则 | 高 |
| **代码片段（Snippets）** | 内置大量代码片段，支持自定义片段，快捷键触发 | 无 | 增加代码片段系统：内置常用查询模板（如创建表、分页查询、行转列等），支持用户自定义 | 中 |

| **查找替换** | Ctrl+F 查找，Ctrl+H 替换，支持正则、全字匹配、区分大小写 | 无内置查找替换对话框（Monaco 自带但可能被覆盖） | 确保 Monaco 的查找替换功能可用（快捷键和菜单调用） | 中 |
| **书签（Bookmarks）** | 可为行添加书签，快速跳转 | 无 | 增加行书签功能：Ctrl+F2 切换书签，F2/Shift+F2 跳转 | 低 |
| **执行历史** | 持久化保存所有执行过的 SQL，支持搜索、收藏、重新执行 | 有 HistoryPanel 组件，但历史仅保存在内存中（页面刷新丢失） | 1. 历史记录持久化到 localStorage；2. 支持收藏常用 SQL；3. 历史记录按连接/数据库分组 | 高 |

### 5.2 查询执行与结果

## 十五、开发建议与技术路线

### 15.1 前端组件拆分建议

当前部分组件文件过大（MainLayout 1015 行、SQLEditor 1316 行、DataTable 1500+ 行），建议：

- `DataTable.tsx` → 拆分为 `DataTableToolbar.tsx`、`DataTableGrid.tsx`、`DataTableFilterPanel.tsx`、`RowEditorModal.tsx`
- `SQLEditor.tsx` → 拆分为 `SqlToolbar.tsx`、`SqlEditorPane.tsx`、`SqlResultPane.tsx`、`SqlHistoryPanel.tsx`
- `MainLayout.tsx` → 将状态管理逻辑提取到 hooks（如 `useConnectionTree.ts`），MainLayout 只负责布局组合

### 15.2 后端 API 需补充

基于当前 `types/api.ts`，后端（Go Sidecar）需要增加以下接口：

- `getProcedures(connectionId, database)` / `getFunctions(connectionId, database)` / `getTriggers(connectionId, database)`
- `getProcedureDetail(connectionId, name, database)` / `getFunctionDetail(...)` / `getTriggerDetail(...)`
- `createProcedure(...)` / `alterProcedure(...)` / `dropProcedure(...)`
- `exportTable(connectionId, tableName, format, options, database)` - 流式导出
- `importTable(connectionId, tableName, format, data, mapping, database)` - 批量导入
- `getTableDDL(connectionId, tableName, database)` - 获取完整建表语句（用于编辑表）
- `executeDDL(connectionId, ddl, database)` - 执行 DDL 语句
- `getForeignKeys` 已存在，但需确保各数据库驱动都实现了该接口

### 15.3 第三方库建议
