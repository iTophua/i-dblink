# 数据库客户端工具 - 需求规格说明书

## 项目概述

### 背景
开发一款类似 Navicat Premium 的跨平台数据库管理工具，支持多种主流数据库及国产达梦数据库，提供可视化数据管理和智能化 SQL 辅助功能。

### 目标
- 打造高性能、轻量级的跨平台桌面数据库客户端
- 支持主流数据库 + 达梦数据库的无缝连接和管理
- 提供直观的可视化界面和智能 SQL 辅助
- 覆盖数据库开发全流程：连接管理、结构设计、数据操作、查询分析、导入导出

---

## 技术架构

### 技术栈

**前端**：Tauri v2 + React 18 + TypeScript + Vite
**后端核心**：Rust (数据库驱动层、连接池管理)
**UI 组件库**：Ant Design 5.x
**图标库**：Ant Design Icons（使用框架自带图标）
**状态管理**：Zustand
**SQL 编辑器**：Monaco Editor
**图表渲染**：D3.js / ECharts
**数据网格**：AG Grid Community

### 架构设计
```
┌─────────────────────────────────────┐
│         UI Layer (React/Vue)        │
│  - 连接管理  - 数据浏览  - SQL 编辑   │
│  - ER 图     - 导入导出  - 设置      │
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│       Tauri Core (Rust)             │

└─────────────────────────────────────┘

┌─────────────────────────────────────┐

```

## 非功能需求

### 性能要求
- 冷启动时间 < 3 秒
- 查询结果加载（1000 行）< 1 秒
- SQL 编辑器响应延迟 < 50ms
- 内存占用 < 500MB（常规使用）
- 支持大文件导入（> 1GB）

### 安全要求
- 密码加密存储（AES-256）
- 使用系统密钥链（Keychain/Keyring）
- SSH 密钥认证支持
- SSL/TLS 加密传输
- 敏感操作确认对话框
- 操作审计日志

### 兼容性要求
- Windows 10/11 (x64, ARM64)
- macOS 11+ (Intel, Apple Silicon)
- Linux (Ubuntu 20.04+, Debian, Fedora)
- 高分屏支持（HiDPI）

### 界面设计要求
- **布局风格**：仿照 Navicat Premium 经典三栏布局
  - 左侧：连接树（支持自定义分组）
  - 中间：对象列表视图
  - 右侧：主工作区（多标签页）
  - 底部：日志/消息面板
- **菜单结构**：传统桌面应用菜单栏（文件/编辑/查看/连接/工具/窗口/帮助）
- **工具栏**：常用操作快捷按钮
- **状态栏**：显示连接状态、执行时间、编码等信息
- **交互习惯**：保持 Navicat 用户的使用习惯，降低学习成本

### 可靠性要求
- 崩溃自动恢复（未保存内容）
- 连接断开自动重连
- 长时间运行稳定性（7x24 小时）
- 异常捕获和友好错误提示

## 风险与挑战

### 技术风险
1. **达梦数据库驱动**：Rust 生态可能缺乏成熟的达梦驱动，需要封装 C 库或自行实现
2. **Oracle 支持**：Oracle 协议复杂，可能需要使用 OCI 库
3. **跨平台一致性**：不同平台的字体渲染、文件路径等差异

### 应对措施
- 提前调研达梦数据库的 API 支持情况
- 考虑使用中间件方案（如通过 JDBC 桥接）
- 建立完善的跨平台测试矩阵

### F. 性能优化策略

**前端优化**：虚拟滚动（AG Grid 内置）、懒加载、代码分割（Vite 自动处理）、Memoization
**后端优化**：连接池管理、异步 IO（Tokio）、零拷贝（Rust 特性）、LTO 编译优化
**内存管理**：RAII（Rust 所有权系统）、流式处理（大数据集分页加载）

### G. 安全考虑

**密码存储**：使用系统密钥链（keyring），不在日志中打印敏感信息
**SQL 注入防护**：参数化查询（sqlx 自动处理）、输入验证、白名单机制
**权限控制**：Tauri 权限系统、IPC 调用验证、文件系统访问限制

### H. 已知风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 达梦驱动不成熟 | 高 | ODBC 桥接方案 |
| Oracle 协议复杂 | 中 | 使用成熟库 rust-oracle |
| 跨平台字体渲染 | 低 | 使用 Web 字体 |
| 大文件导入性能 | 中 | 流式处理 + 进度条 |
